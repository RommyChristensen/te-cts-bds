# Render Deployment Guide with PostgreSQL

## Step-by-Step Instructions

### Step 1: Prepare Your Repository
1. Push all changes to GitHub
2. Ensure your repository is public or connected to Render

### Step 2: Create PostgreSQL Database on Render
1. Go to Render Dashboard
2. Click "New +" → "PostgreSQL"
3. Configure:
   - Name: `game-database`
   - Database Name: `game_database`
   - User: `game_user`
   - Plan: Free
   - Region: Oregon (or nearest)
4. Click "Create Database"

### Step 3: Create Web Service
1. Click "New +" → "Web Service"
2. Connect your GitHub repository
3. Configure:
   - Name: `interactive-game-website`
   - Runtime: Node
   - Plan: Free
   - Region: Same as database
   - Build Command: `npm install && npm run migrate`
   - Start Command: `npm start`
   - Health Check Path: `/`

### Step 4: Add Environment Variables
In your Web Service settings, add these environment variables:

```env
NODE_ENV=production
PORT=10000
SESSION_SECRET=<auto-generated>
DATABASE_URL=<from-database-connection-string>
```

### Step 5: Connect Database
1. In your Web Service settings, scroll to "Databases"
2. Click "Connect existing database"
3. Select your `game-database`
4. Render will automatically add `DATABASE_URL`

### Step 6: Deploy
1. Click "Create Web Service"
2. Render will automatically:
   - Install dependencies
   - Run database migrations
   - Start your application

### Step 7: Verify Deployment
1. Check the deployment logs
2. Visit your app URL
3. Test login and game functionality

## Important Notes

### Database Migration
The `npm run migrate` command will automatically:
- Create all tables
- Seed initial data
- Set up foreign keys and triggers

### Environment Variables
Render automatically provides:
- `DATABASE_URL` - PostgreSQL connection string
- Database connection details

### Troubleshooting
If migrations fail:
1. Check the build logs
2. Ensure database is created before web service
3. Verify database permissions

### Database Access
To access your database on Render:
1. Go to PostgreSQL dashboard
2. Click on your database
3. Use "External Connection" or "Shell" access

## Alternative: Use Render Blueprint

If you prefer blueprint-based deployment, create a `render.yaml` file:

```yaml
services:
  - type: web
    name: interactive-game-website
    runtime: node
    plan: free
    buildCommand: npm install && npm run migrate
    startCommand: npm start
    envVars:
      - key: NODE_ENV
        value: production
      - key: PORT
        value: 10000
      - key: SESSION_SECRET
        generateValue: true
    databases:
      - name: game_database
        databaseName: game_database
```

## Post-Deployment Checklist

- [ ] Application loads successfully
- [ ] Database migrations completed
- [ ] Users can login
- [ ] Admin dashboard works
- [ ] Game creation/editing works
- [ ] Real-time features work
- [ ] Currency system works

## Production Considerations

1. **Database Backups**: Enable automatic backups in Render
2. **Monitoring**: Set up alerts for database and app performance
3. **Scaling**: Upgrade plans if needed for more users
4. **Security**: Use SSL connections (automatic on Render)
